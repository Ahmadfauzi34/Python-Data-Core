## 2024-05-18 - [O(N^2) Vector Similarity Bottleneck] **Learning:** The pairwise distance calculation (`np.mean(np.cos(a-b))`) in `FHRRTopologicalLayer._ensure_distance_matrix` using nested Python loops is a massive CPU bottleneck, taking ~17.9 seconds for just 500 tokens. **Action:** Refactored into highly vectorized matrix multiplication using trig identity `cos(a-b) = cos(a)cos(b) + sin(a)sin(b)`. Brought runtime to ~0.15s (>100x speedup). Always look for trig identities to map circular operations to standard BLAS routines when vectorizing HRR/FHRR models.
## 2024-05-18 - [O(1) Vocabulary Lookup] **Learning:** Extensive use of `list.index()` across the engine (for roles and tokens) caused hidden O(N) performance degradations, particularly when parsing large queries or iterating topologies. **Action:** Maintained shadow dictionaries `_token_name_to_idx` and `_role_name_to_idx` alongside the lists to drop retrieval latency from ~0.50s to ~0.003s in micro-benchmarks. Always back frequently accessed categorical lists with hash maps in core loop architectures.
